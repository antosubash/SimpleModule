# Frontend Integration (React + Inertia.js)

## How It Works

1. ASP.NET endpoint calls `Inertia.Render("Module/Page", props)`
2. Inertia middleware renders static HTML shell with embedded JSON props
3. React ClientApp dynamically imports module's `pages.js` bundle
4. Component hydrates with server-provided props

## Pages Registry (CRITICAL)

Every `IViewEndpoint` with `Inertia.Render()` **must** have a matching entry in `Pages/index.ts`:

```typescript
// modules/{Name}/src/SimpleModule.{Name}/Pages/index.ts
export const pages: Record<string, unknown> = {
    '{Name}/Browse': () => import('../Views/Browse'),
    '{Name}/Manage': () => import('../Views/Manage'),
    '{Name}/Create': () => import('../Views/Create'),
    '{Name}/Edit': () => import('../Views/Edit'),
};
```

The key must match the first argument of `Inertia.Render()` exactly. If you forget:
- Server-side works fine
- Client-side silently 404s (no error in console)

**Validate:** `npm run validate-pages`

## React View Components

```tsx
// modules/{Name}/src/SimpleModule.{Name}/Views/Browse.tsx
import { PageShell } from '@simplemodule/ui/components';
import type { Product } from '../types';

interface BrowseProps {
    products: Product[];
}

export default function Browse({ products }: BrowseProps) {
    return (
        <PageShell title="Products">
            {products.map((p) => (
                <div key={p.id}>{p.name} - ${p.price}</div>
            ))}
        </PageShell>
    );
}
```

## Navigation with Inertia Router

```tsx
import { router } from '@inertiajs/react';

// Navigate
router.get('/products/manage');

// Form submit
router.post('/products', formData);

// Delete
router.delete(`/products/${id}`);
```

## TypeScript Types

Types are auto-generated from `[Dto]` C# classes:

```typescript
// types.ts (auto-generated by source generator + extract-ts-types.mjs)
export interface Product {
    id: number;
    name: string;
    price: number;
}
```

Generate types: `npm run generate:types`

## Vite Configuration

Each module uses a minimal Vite config:

```typescript
// vite.config.ts
import { defineModuleConfig } from '@simplemodule/client/module';
export default defineModuleConfig(__dirname);
```

This configures library mode, externalizes React/React-DOM/@inertiajs/react, and outputs `{Name}.pages.js` to `wwwroot/`.

## Inertia Shared Data

Available server-side for data shared across all responses in a request:

```csharp
InertiaSharedData.Set("flash", new { message = "Saved!" });
InertiaSharedData.Get<string>("key");
```

## Props Serialization

- Props serialize to **camelCase** automatically
- Endpoint props override shared data
- Anonymous objects are the standard pattern: `new { products, totalCount }`

## Linting & Formatting

npm run check        # lint + format check
